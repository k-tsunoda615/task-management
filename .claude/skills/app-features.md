# タスク管理アプリ - 機能一覧

## 1. 認証・ユーザー管理

### 1.1 メール/パスワード認証
- メールアドレスとパスワードによるサインアップ・ログイン
- Supabase Auth を使用
- メール確認フロー (`/confirm` ページ)
- パスワードリセット機能

### 1.2 匿名ログイン (ゲストモード)
- アカウント登録不要でタスク操作を体験可能
- Supabase Anonymous Auth を利用
- 永続ユーザーへのアップグレードフロー (メール登録 + パスワード設定)

### 1.3 Google OAuth
- Google アカウントによるソーシャルログイン
- Supabase OAuth プロバイダ連携

### 1.4 認証ミドルウェア
- 未認証ユーザーは `/auth` にリダイレクト
- 認証済みユーザーは `/board` にリダイレクト
- ルートごとの認証要否をミドルウェアで制御

## 2. タスク管理 (CRUD)

### 2.1 タスク作成
- タイトル (必須)、メモ (Markdown 対応)、ステータス、タグ、作業時間を指定
- モーダルダイアログから作成
- 作成時に `sort_order` を自動計算

### 2.2 タスク編集
- タイトル・メモ・ステータス・完了フラグ・公開/非公開の変更
- 編集モーダルまたはタスク詳細ページ (`/note/[id]`) から実行
- Markdown プレビュー機能付き

### 2.3 タスク削除
- 確認ダイアログ付き削除
- リストビューでは一括削除 (チェックボックス選択) に対応
- 関連する `todo_tags` も連動削除

### 2.4 タスクステータス
- **Priority** (重要): 優先的に取り組むタスク
- **Next** (次にやる): 次に着手予定のタスク
- **Archived** (完了/保留): 完了済みまたは保留中のタスク
- ステータス変更はドラッグ&ドロップまたは手動切り替え

### 2.5 ファイル添付
- タスクにファイルをアップロード・添付
- Supabase Storage (`task-assets` バケット) で管理
- 対応形式: 画像, 動画, 音声, PDF, テキスト, Markdown, CSV, JSON
- 最大サイズ: 100MB

## 3. ビュー

### 3.1 カンバンボード (`/board`)
- 3 カラム構成 (Priority / Next / Archived)
- カラム間ドラッグ&ドロップでステータス変更
- カラム内ドラッグ&ドロップで並び順変更
- レイアウト切り替え (4:1 / 3:2 / 1:1 / 1カラム)
- タグ表示 / タイマー表示のトグル
- カード上でのクイックタイマー操作

### 3.2 リストビュー (`/list`)
- テーブル形式でのタスク一覧表示
- カラムソート (タイトル, ステータス, 更新日, 作業時間 等)
- 検索 (タイトル・メモのテキスト検索)
- フィルタ (タグ, ステータス, 公開/非公開)
- ドラッグで並び替え
- チェックボックスによる一括選択・一括削除
- 合計作業時間の表示

### 3.3 タスク詳細 (`/note/[id]`)
- タスクの全情報を表示・編集
- Markdown メモのリアルタイムプレビュー
- タイマーの開始/停止
- ステータス変更・完了フラグ切り替え
- 添付ファイル管理

### 3.4 アナリティクス (`/analytics`)
- 期間別フィルタ (全期間, 7日, 30日, 90日 等)
- ステータス分布チャート (Chart.js)
- タグ分布チャート
- 作業時間分布チャート
- 最近のアクティビティテーブル
- 完了タスクの表示/非表示切り替え

## 4. タグ管理

### 4.1 タグ CRUD
- タグの作成・編集・削除
- カスタムカラー設定
- `TagManageModal` からの操作

### 4.2 タグの並び順
- ドラッグ&ドロップで順序変更
- `sort_order` による 100 刻みの順序管理

### 4.3 タグフィルタリング
- サイドバーからタグでタスクを絞り込み
- 複数タグの AND/OR フィルタ

## 5. タイムトラッキング

### 5.1 タイマー機能
- タスクごとに開始/停止可能なストップウォッチ
- `requestAnimationFrame` ベースの滑らかな更新
- ブラウザタブタイトルに経過時間を表示
- 累積作業時間 (`total_time`) を秒単位で管理

### 5.2 タイマーガード
- タイマー稼働中のページ遷移を確認ダイアログでブロック
- グローバルナビゲーションガード (`timer-guard.global.ts` プラグイン)
- データ損失防止

### 5.3 手動時間入力
- `hh:mm:ss` 形式での作業時間手動入力
- タスク作成時・編集時に設定可能

## 6. AI 機能

### 6.1 AI チャット
- Google Gemini API を利用したチャットインターフェース
- サーバーサイドプロキシ (`/api/gemini/chat`) 経由で API キーを保護
- 会話履歴の保持
- モデル選択機能

### 6.2 AI タイトル生成
- タスクメモの内容から適切なタイトルを自動生成
- `useAITitleGenerator()` composable で実装

## 7. サイドバー・ナビゲーション

### 7.1 サイドバー
- 折りたたみ可能なサイドメニュー
- 新規タスク作成ボタン
- タグフィルタ・タグ管理へのアクセス
- 表示切り替え (公開/非公開/全て)
- ビュー切り替えリンク (Board / List / Analytics)

### 7.2 レスポンシブデザイン
- モバイル/タブレット/デスクトップ対応
- ブレークポイントに応じたレイアウト切り替え

## 8. オンボーディング

### 8.1 初回ログイン時のサンプルデータ
- 新規ユーザーにサンプルタグとサンプルタスクを自動投入
- `useOnboardingSetup()` composable で実装
- アプリの使い方を体験できるデモデータ

## 9. 管理者ダッシュボード (`/admin`)

### 9.1 ユーザーメトリクス
- ユーザー一覧 (メール, 表示名)
- ユーザーごとの Todo 件数
- ストレージ使用量
- Supabase ビュー `admin_user_metrics` からデータ取得

## 10. イベント計測・アナリティクス

### 10.1 Google Analytics 連携
- Google Tag Manager 経由で GA4 にイベント送信
- ページビュー自動計測

### 10.2 計測イベント一覧
| イベント | トリガー |
|---------|---------|
| タスク作成 | `trackTaskCreated()` |
| タスク更新 | `trackTaskUpdated()` |
| タスク削除 | `trackTaskDeleted()` |
| ステータス変更 | `trackTaskStatusChanged()` |
| タイマー開始 | `trackTimerStarted()` |
| タイマー停止 | `trackTimerStopped()` |
| レイアウト変更 | `trackLayoutChanged()` |
| 検索実行 | `trackSearch()` |
| タグフィルタ | `trackTagFiltered()` |

## 11. データ同期

### 11.1 自動リフレッシュ
- ウィンドウフォーカス時に自動同期
- `document.visibilitychange` イベントでの同期
- 定期リフレッシュ (デフォルト 30 分間隔、設定変更可能)
- `useTodoSync()` composable で実装

### 11.2 楽観的更新
- UI をまず即座に更新し、バックグラウンドで DB に反映
- 失敗時はロールバックまたは再取得

## 12. ランディングページ

### 12.1 LP (A/B テスト対応)
- `/lp/a` と `/lp/b` の 2 パターンを用意
- 共通ヘッダー・フッター
- GA4 で効果測定可能な設計

## 13. セキュリティ

### 13.1 認証・認可
- Supabase Auth によるセッション管理
- Row Level Security (RLS) による DB レベルのアクセス制御
- サーバーサイド API で API キーを保護

### 13.2 入力サニタイズ
- DOMPurify による HTML サニタイズ (Markdown → HTML 変換後)
- Gemini API メッセージの文字数制限 (10,000 文字)

### 13.3 ファイルアップロード
- MIME タイプ制限
- ファイルサイズ制限 (100MB)
